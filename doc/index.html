Module playluaengine

PlayLua Engine is a Lua-scriptable Game Engine to create 3D games (based on OpenGL) or interactive Windows applications with Arduino.


INTRODUCTION

The contents of this documentation are sectioned according to the Engine components.

ple is the table that contains all the functions and objects of the PlayLua Engine, and it is created by invoking require("playluaengine").

The load of the module playluaengine also allows loading other third-party modules that are useful as the iup (GUI), luagl (for OpenGL) and penlight (for class).

The application framework of PlayLua Engine is similar to iup, with the following differences:

  • The application dialog is ple.dialog and its execution runs on account of ple:play().
  • ple.canvas (glcanvasbox) is the container-canvas that will be used for videogames, and like any canvas, it has events on which tasks such as initialization, resizing, updating (draw elements), etc., can be performed.

All the "Windows Mode Examples" that integrate PlayLua are based on PlayLua Engine, and are a good guide to using the components of the PlayLua Engine.

BODIES

ple.bodies
This object represents the Bodies, that is to say the Physics Engine. Often the game objects (Formatas and Animatas) are linked to Bodies to guide their movement and detect collisions. To access the i-th Body, invoke ple.bodies[i], which returns an TBody object. The ple.bodies object has some fields (abstract methods) to be defined by the programmer, and can be invoked when physical events occur. Bellow, the parameters of such methods and methods will be mentioned;

b: (int) index of the Body in study.
bk: (string) type of the Game Object linked to the Body in study, possible values: "NONE", "FORMATA", "ANIMATA".
bi: (int) index of the Game Object linked to the Body in study.
t: (int) index of the Test Body.
tk: (string) type of the Game Object linked to the Test Body, possible values: "NONE", "FORMATA", "ANIMATA".
ti: (int) index of the Game Object linked to the Test Body.
p: (string) plane of the impact, possible values: "X", "Y", "Z".
x,y,z: (number) the coordinates of the position where the event occurs.

Fields:

  • onCollisionUp func (b, bk, bi, t, tk, ti, x, y, z)
  • onCollisionDown func (b, bk, bi, t, tk, ti, x, y, z)
  • onCollisionSide func (b, bk, bi, t, tk, ti, x, y, z)
  • onGround func (b, bk, bi, t, tk, ti, x, y, z)
  • onRunEnter func ()
  • onRunExit func ()
  • onAscend func ()
  • onDescend func ()
  • onBulletDispatch func (b, bk, bi, x, y, z)
  • onBulletImpact func (b, bk, bi, p, x, y, z)

Usage:

    ple.bodies[0] -- returns the Body with index 0.
ple.bodies:add (...)
Add a Body. Note: All Bodies created with this method are not linked to any game object (Formata or Animata), so in an event, the type of game object to which it is linked will be "NONE".

-- Add a Body sensitive to gravity and defined by: "c:/body.smd"

Draw all Bodies.

ANIMATIONS

ple.animations
This object represents the Animations (OBJ animations). The Animations can not be drawn, they only serve to construct the Animatas, which can be drawn, can be linked with Bodies, etc. The OBJ files of an OBJ animation are read in order, and in that order the frames are assigned, although starting from 0, so the first file will be taken as frame 0, the second as frame 1, and so on. You must consider this enumeration because you must specify frames for each movement in the attrs.lua file.

About the OBJ animations: The OBJ animations should be built with Blender (> 2.6) through the OBJ exporter, starting from frame 0, frame by frame to the end. During modeling, be sure to triangulate all faces to avoid inconsistencies in the interpolations. In the OBJ exporter options only select: -Z Forward, Y Up, Animation, Apply Modifiers, Write Normals, Include UVs, Write Materials, Triangulate Faces and Objects as OBJ Objects.

About the attrs.lua file: This file is as important as any OBJ file, since there you include the information of the movements, such as the name and the frames that correspond to it. You can also include other attributes such as: Speed, Eye Position, etc. (See an attrs.lua file of an example animation).
